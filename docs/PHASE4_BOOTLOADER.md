# Phase 4: Bootloader

2O9 declares the bootloader in `2O9.nix` under `boot.loader`. `209 apply`
installs it and generates a menu with one entry per 2O9 generation. Two
backends are supported: grub (BIOS and UEFI) and systemd-boot (UEFI only).
You enable exactly one. Enabling both is an error.

## Config schema

The `boot` block carries the kernel selection, module lists, the loader
choice, and the file systems (the latter is shared with the fstab
module).

```nix
{
  boot = {
    kernel = "linux";           # "linux", "linux-lts", "linux-zen"
    kernelModules = [ "nvme" "ext4" ];
    initrdModules = [ "nvme" "ext4" ];

    loader = {
      grub = {
        enable = true;
        device = "/dev/sda";       # BIOS target. "nodev" for UEFI.
        efiSupport = false;
        efiInstallAsRemovable = false;
        extraEntries = ''
          menuentry "Windows" {
            set root=(hd0,1)
            chainloader +1
          }
        '';
        extraConfig = ''
          set timeout=5
        '';
        useOSProber = true;
      };

      # OR systemd-boot (UEFI only):
      # systemd-boot = {
      #   enable = true;
      #   consoleMode = "max";
      #   editor = false;
      #   extraEntries = ''
      #     title Arch Linux (LTS)
      #     linux /vmlinuz-linux-lts
      #     initrd /initramfs-linux-lts.img
      #     options root=LABEL=arch rw
      #   '';
      # };
    };

    fileSystems = {
      "/" = { device = "/dev/sda2"; fsType = "ext4"; };
      "/boot" = { device = "/dev/sda1"; fsType = "vfat"; };
    };
  };
}
```

## grub

`boot.loader.grub` configures grub. Set `enable = true` and pick a
`device` for BIOS installs, or set `device = "nodev"` (or
`efiSupport = true`) for UEFI installs.

### BIOS install

```nix
boot.loader.grub = {
  enable = true;
  device = "/dev/sda";
};
```

`209 apply` runs `grub-install --target=i386-pc /dev/sda`. `device` is
required for BIOS installs; omitting it is an error.

### UEFI install

```nix
boot.loader.grub = {
  enable = true;
  device = "nodev";
  efiSupport = true;
  efiInstallAsRemovable = false;  # add --removable when true
};
```

`209 apply` runs
`grub-install --target=x86_64-efi --efi-directory=/boot/EFI --bootloader-id=2O9`.
With `efiInstallAsRemovable = true` it appends `--removable` so the
binary lands at `/boot/EFI/BOOT/BOOTX64.EFI` for firmwares that will not
read the bootloader-id path.

### grub.cfg generation

`209 apply` writes `/boot/grub/grub.cfg`. Two paths:

- With `useOSProber = true`, it runs
  `GRUB_DISABLE_OS_PROBER=false grub-mkconfig -o /boot/grub/grub.cfg`
  and then appends the 2O9 generation entries. This picks up other
  installed OSes (Windows, other Linux installs).
- Without `useOSProber`, it writes the file from scratch with
  `set default=0`, `set timeout=<N>` (parsed from `extraConfig` or 5),
  one `menuentry` per generation, then `extraEntries` and `extraConfig`
  verbatim.

Each generation's menuentry looks like:

```
menuentry "2O9 Generation #42 (2024-09-01)" {
    linux /boot/vmlinuz-linux root=/dev/sda2 rw init=/nix/store/<hash>-hoshizora-<ver>/init 2O9_GENERATION=42
    initrd /boot/initramfs-linux.img
}
```

The current generation is the default entry (position 0). Older
generations follow in descending order so you can pick an earlier one
from the menu to roll back at boot.

`extraEntries` is appended after the 2O9 entries. Use it for things
like a Windows chainloader entry. `extraConfig` is appended after that;
any `set timeout=N` in `extraConfig` overrides the timeout 2O9 wrote
because GRUB takes the last `set` it sees.

## systemd-boot

`boot.loader.systemd-boot` configures systemd-boot. UEFI only.

```nix
boot.loader.systemd-boot = {
  enable = true;
  consoleMode = "max";   # "auto", "max", "keep", or a number
  editor = false;        # disable the kernel command-line editor at boot
};
```

`209 apply` runs `bootctl install` if
`/boot/EFI/systemd/systemd-bootx64.efi` does not exist, or `bootctl
update` if it does. It writes `/boot/loader/loader.conf`:

```
# Generated by 2O9. Do not edit; run `209 apply` to regenerate.
default 2O9-42
console-mode max
editor no
timeout 5
```

The `default` line points at the current generation's entry file.

For each generation, `209 apply` writes
`/boot/loader/entries/2O9-<N>.conf`:

```
title   2O9 Generation #42
linux   /vmlinuz-linux
initrd  /initramfs-linux.img
options root=/dev/sda2 rw init=/nix/store/<hash>-hoshizora-<ver>/init 2O9_GENERATION=42
```

Old generation entry files stay on disk so the boot menu keeps showing
them. They are rewritten on every `209 apply` to stay in sync with the
manifest.

## Kernel and initrd

`boot.kernel` selects the kernel package. The default is `"linux"`.
The declared kernel must be installed in the current generation's
package list. If it is missing, `209 apply` prints:

```
bootloader: declared kernel 'linux' not in current generation's package list; run `209 -S linux` first. Skipping initrd.
```

The bootloader config (grub.cfg or entry files) is still written, but
`mkinitcpio` is skipped. Install the kernel with `209 -S linux` (or
`linux-lts`, `linux-zen`, etc.) and run `209 apply` again.

When the kernel is present, `209 apply` runs `mkinitcpio -P` to
regenerate all initramfs images. If `boot.initrdModules` is set, it
writes a temporary `mkinitcpio.conf` with those modules in `MODULES=`
and passes it via `mkinitcpio -c <temp> -P`. If `mkinitcpio` is not
installed, 2O9 prints a warning and skips initrd regeneration.

`boot.kernelModules` is recorded in the manifest for reference. The
initrd only gets `boot.initrdModules` because mkinitcpio builds the
initramfs from those; kernelModules are loaded by the running kernel
later in boot.

## Root device

The `root=` option in each menu entry comes from
`boot.fileSystems["/"].device`. If that is missing, 2O9 falls back to
`/dev/sda2` and prints a warning. Declare the root file system in the
`boot.fileSystems` block to avoid the fallback.

## Generation enumeration

`209 apply` walks `<db_root>/generations/`. Each subdirectory is a
generation number. Its `manifest.json` supplies the package list; 2O9
looks for the first `linux*` package (excluding `linux-headers`,
`linux-firmware`, and similar non-kernel packages) to find the kernel
name, and the `hoshizora` package to find the init path. The file's
mtime is the menu date.

The current generation (per `<db_root>/current`) is the default entry.
If `db_root` is NULL, the bootloader is installed but no generation
entries are written.

## Switching bootloaders

If the previous generation's manifest enabled one bootloader and the
current enables the other, `209 apply` prints a warning naming the old
bootloader's files that are left in place:

```
bootloader: WARNING switched from grub to systemd-boot; old /boot/grub left in place (clean up manually if desired)
```

2O9 does not delete `/boot/grub`, `/boot/loader/entries`, or
`/boot/EFI/systemd` automatically. Remove them by hand if you want a
clean switch.

## Edge cases

- Neither grub nor systemd-boot enabled: `209 apply` prints `bootloader:
  no bootloader configured` and returns success.
- Both enabled: `209 apply` errors out with `cannot enable both grub
  and systemd-boot`.
- grub BIOS install with no `device`: `209 apply` errors out with
  `grub.device required for BIOS install`.
- `/boot/grub/` missing: 2O9 creates it before writing `grub.cfg`.
- `mkinitcpio` missing: 2O9 warns and skips initrd regeneration.
- `grub-install` or `bootctl` missing: 2O9 reports the missing binary
  and returns non-zero.

## Permissions

`bootloader_apply` checks `getuid() == 0` and returns an error
otherwise. It writes under `/boot`, runs `grub-install`, `bootctl`,
`grub-mkconfig`, and `mkinitcpio`, all of which need root. Run
`sudo 209 apply`.

## Examples

### BIOS grub with OS prober

```nix
{
  boot = {
    kernel = "linux";
    loader.grub = {
      enable = true;
      device = "/dev/sda";
      useOSProber = true;
      extraConfig = ''
        set timeout=10
      '';
    };
    fileSystems."/" = { device = "/dev/sda2"; fsType = "ext4"; };
  };
}
```

### UEFI systemd-boot

```nix
{
  boot = {
    kernel = "linux-lts";
    initrdModules = [ "nvme" "ext4" "usb_storage" ];
    loader.systemd-boot = {
      enable = true;
      consoleMode = "max";
      editor = false;
    };
    fileSystems = {
      "/"     = { device = "/dev/nvme0n1p2"; fsType = "ext4"; };
      "/boot" = { device = "/dev/nvme0n1p1"; fsType = "vfat"; };
    };
  };
}
```

### UEFI grub with a Windows chainloader entry

```nix
{
  boot = {
    kernel = "linux-zen";
    loader.grub = {
      enable = true;
      device = "nodev";
      efiSupport = true;
      extraEntries = ''
        menuentry "Windows" {
          insmod part_gpt
          set root='hd0,gpt1'
          chainloader /EFI/Microsoft/Boot/bootmgfw.efi
        }
      '';
    };
    fileSystems = {
      "/"     = { device = "/dev/sda2"; fsType = "ext4"; };
      "/boot" = { device = "/dev/sda1"; fsType = "vfat"; };
    };
  };
}
```
